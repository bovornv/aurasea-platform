# Pages

Page components for the hospitality application.

## Responsibilities

- Define page layouts and routes
- Compose components for each page
- Handle page-level state
- Navigate between pages

## Boundaries

- **DO NOT** implement business logic
- **DO NOT** make direct API calls
- **DO** use services layer for data
- **DO** use shared components from packages/ui

## Pages

- `login/` - Authentication page
- `home/` - Dashboard home page
- `alert/` - Alerts and notifications page
- `overview/` - Business overview page
- `scenario/` - Scenario planning page
- `history/` - Decision history page
- `settings/` - Application settings page
