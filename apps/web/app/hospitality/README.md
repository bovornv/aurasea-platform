# Dashboard MVP - Phase 4

## Overview

Phase 4 MVP provides a minimal, production-quality web interface for hospitality businesses to view alerts generated by SME OS.

**Key Principle**: This is UI + wiring only. No business logic, no changes to core/sme-os.

## File Structure

```
app/hospitality/
├── page.tsx                    # Dashboard (/hospitality)
│   └── High-level summary, alert counts by severity, last updated
│
├── alerts/
│   ├── page.tsx               # Alert List (/hospitality/alerts)
│   │   └── Table/list of all alerts with key columns
│   │
│   └── [id]/
│       └── page.tsx           # Alert Detail (/hospitality/alerts/[id])
│       └── Full alert details, explanation, key facts
│
└── README.md                   # This file
```

## Data Flow

```
Hospitality Data (mock/seeded)
    ↓
Adapter Layer (app/adapters/hospitality-adapter.ts)
    ↓
SME OS Input Contract (core/sme-os/contracts/inputs.ts)
    ↓
SME OS (core/sme-os/) - NO CHANGES ALLOWED
    ↓
SME OS Alert Contract (core/sme-os/contracts/alerts.ts)
    ↓
Adapter Layer (translate back to hospitality context)
    ↓
UI Components (render alerts)
```

## Adapter Responsibilities

The adapter layer (`app/adapters/hospitality-adapter.ts`) handles:

1. **Input Translation**: 
   - Maps hospitality financial data (cash balance, inflows, outflows, fixed costs)
   - Converts to SME OS `InputContract` format

2. **Output Translation**:
   - Maps SME OS `AlertContract` back to hospitality-friendly format
   - Adds hospitality-specific labels and context

## Service Layer

Services (`app/services/sme-os-service.ts`) are the ONLY boundary to SME OS:
- Call SME OS evaluators
- Return alert outputs
- Handle errors

## Pages

### 1. Dashboard (`/hospitality`)
- Alert counts by severity (Info / Warning / Critical)
- Last updated timestamp
- High-level summary

### 2. Alert List (`/hospitality/alerts`)
- Table or list view
- Columns: Alert name, Severity, Confidence %, Time horizon, Last evaluated
- Click → Navigate to Alert Detail

### 3. Alert Detail (`/hospitality/alerts/[id]`)
- Alert name + severity badge
- Confidence score (labeled as descriptive)
- Explanation section (SME OS explainer output)
- Key facts (dates, days of coverage, pattern deviation)
- Explicit note: "This system explains risk. It does not give advice."

## Constraints

- ✅ Web UI pages and components
- ✅ Data adapter layer
- ✅ API endpoints to fetch alerts
- ✅ Routing, layout, navigation

- ❌ NO changes to core/sme-os logic
- ❌ NO new SME OS alerts
- ❌ NO recommendations or "what to do next"
- ❌ NO ML or forecasting beyond SME OS

## Design

- Style: Stripe + Linear (calm, professional, decision-grade)
- No gamification
- No charts unless directly supporting explanation
- Read-only (no actions, no recommendations)
